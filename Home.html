Project Description
Inclusion of geo-data has become one of the prime requirements in majority of the applications, irrespective of their domain – mobile or enterprise class applications. Creating an API that can satisfy even minimum requirements like querying of continents, countries, states in a country etc. can be an arduous task and finding the right set of data that gives you just the right resolution can be even harder.

Thankfully, the good folks at Geonames.org understand this and give away the geo-spatial data which should prove sufficient for majority of general purpose needs. They also their data in the form of free to consume API, which understandably comes capped in terms of number of calls that can be made.

This project uses the data dump made available (Can be downloaded from Geonames.org) from a SQL Server database and gives you a RESTful API surface that you can consume in any application, built using any technology, so long it can consume REST services. So you get the independence of tailoring the solution to your needs, hosting it in your environment and controlling it the way you want. The existing programming in this project gives you a good jump-start.

Geonames Web API is a RESTful API built using Web API v2. The project at the moment consists of the services that can be used to query information about the following:

  1. Continents
  2. Countries
  3. States
  4. Cities
  5. Timezones
  6. Languages
  7. Postal codes

The services offered in the project are all, at the moment HttpGet type.

NOTE: The project uses the data as provided by Geonames.org, The data as given by Geonames comes encoded in UTF-8 encoding and needs conversion into UTF-16 encoding, compatible with SQL Server. For converting character encoding you can use characterencodingconverter.codeplex.com, a free character encoding converter that can handle multi-gigabyte files and all the encodings as supported by the .NET framework.

The project download consists of the following:

  1. SQL scripts to help you create the database, tables, stored procedures and bulk insert data from the files with UTF-16 character encoding to the tables.
  2. Solution consisting of projects that make up the domain, repository, data access layer and Web API itself.
